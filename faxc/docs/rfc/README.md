# RFCs (Request for Comments)

## Active RFCs

None yet. RFCs will be added as the language evolves.

## RFC Process

1. **Draft**: Write initial proposal in `drafts/`
2. **Discussion**: Open GitHub discussion for feedback
3. **Review**: Core team reviews and requests changes
4. **Accept**: Merge to `accepted/` directory
5. **Implement**: Track implementation in issues

## Template

```markdown
# RFC: [Title]

## Summary
One paragraph explanation.

## Motivation
Why are we doing this?

## Detailed Design
Technical details.

## Drawbacks
Potential issues.

## Alternatives
Other approaches considered.

## Unresolved Questions
What needs more discussion?
```
